##   能够使用中间件检查用户权限
```
1、时机
	在用户登录成功后，知道用户的角色，通过模型关联，角色关联到节点表，知道用户有哪些权限，权限放到session中
2、菜单显示
	菜单对应的节点，节点表中，以is_menu字段为依据 0 不是菜单  1菜单
	根据刚刚用户所对应的权限节点 in 节点表 is_menu='1' 得到了用户的菜单
	写了一个递归方法 模型中Base.php文件中 让数据层级显示
		id 1
		name 管理员管理
		sub
			id 2
			name 用户管理
3、后台访问权限
	页面限制访问
        限制是什么：
            限制了你每次打开的页面，在每次打开的页面中，进行用户权限的判断，判断此用户是否有打开此页面的权限
        什么是每次打开页面都会执行的文件呢？
            路由  ---->  中间件 ---> 控制器@方法(页面)
        之前在防用户翻墙的时候创建一个中间件，在此中间件中继续加代码

            根据当前登录的用户对应的角色，取出对应的节点，后对应的路由别名，并且还要添加一些不需要认证用户或路由
            根据当前路由和角色所拥有的路由别名比对，有则访问，无则拒绝
	页面按钮限制
		在什么地方去限制按钮
			在模板中 @if	在程序写
		推荐写在程序中
			本着组件化使用，是谁的，写在谁的程序
				程序 控制、模型，写在哪呢？
					写在模型中，如果有数据的传递
					如果指写在自己的模型的，复用性就没有了，所以我们想到了多继承
					php5.4之后，提供trait 可以用它来实现多继承，实现在代码复用
```
